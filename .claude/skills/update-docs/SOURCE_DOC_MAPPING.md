# Source-to-Doc Mapping

Maps pipecat-cli source files to their documentation pages. Source paths are relative to `src/pipecat_cli/`. Doc paths are relative to the docs repo root.

CLI documentation lives at `api-reference/cli/` in the docs repo.

## Direct mappings

| Source path | Doc page |
|---|---|
| `commands/init.py` | `api-reference/cli/init.mdx` |
| `main.py` | `api-reference/cli/overview.mdx` |
| `registry/service_metadata.py` | `api-reference/cli/init.mdx` |
| `prompts/questions.py` | `api-reference/cli/init.mdx` |
| `config_validator.py` | `api-reference/cli/init.mdx` |
| `generators/project.py` | `api-reference/cli/init.mdx` |
| `templates/**` | `api-reference/cli/init.mdx` |

Multiple source files map to the same doc page because `init` is the primary command and its behavior is shaped by the registry, prompt flow, config validation, project generation, and templates together.

## Skip list

These files should never trigger doc updates.

| Pattern | Reason |
|---|---|
| `registry/_configs.py` | Auto-generated by `scripts/update_registry.py` |
| `registry/_imports.py` | Auto-generated by `scripts/update_registry.py` |
| `registry/service_loader.py` | Internal query logic, not user-facing |
| `__init__.py` | Module init files |

## Notes

- The `cloud` and `tail` commands come from external packages (`pipecatcloud`, `pipecat-ai-tail`) installed via entry points. Changes to those commands won't appear in this repo's diffs.
- The `api-reference/cli/` section in the docs repo contains ~11 pages. Most source changes in this repo affect the `init.mdx` page since that's the main scaffolding command.
